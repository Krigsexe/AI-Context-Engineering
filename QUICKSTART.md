# 🚀 ODIN Quick Start Guide

Welcome! This guide will have you up and running with ODIN in just 5 minutes.

## 🎯 What You'll Build

By the end of this guide, you'll have an AI assistant that can:
- Write code for you
- Fix bugs automatically
- Document everything it does
- Learn from your feedback

## 📋 Prerequisites

You only need:
- An AI tool (ChatGPT, Claude, or any local LLM)
- A text editor or IDE (VSCode, Notepad++, anything works!)

That's it! No coding experience required.

## 🏃 Let's Get Started!

### Step 1: Get ODIN (30 seconds)

```bash
git clone https://github.com/Krigsexe/AI-Context-Engineering.git
cd AI-Context-Engineering
```

Or simply [download as ZIP](https://github.com/Krigsexe/AI-Context-Engineering/archive/refs/heads/main.zip) if you don't have Git.

### Step 2: Load the ODIN Prompt (1 minute)

1. Open the file `prompts/ODIN.md`
2. Copy its entire content
3. Paste it into your AI assistant (ChatGPT, Claude, etc.)
4. Add this message: "You are now ODIN. Please confirm you understand your role."

### Step 3: Your First Project (3 minutes)

Try this simple example:

```
Create a simple to-do list web app with these features:
- Add tasks
- Mark tasks as complete
- Delete tasks
- Save tasks locally

Please create all necessary files.
```

The AI will:
1. Create the HTML, CSS, and JavaScript files
2. Document what each part does
3. Ask for your feedback

### Step 4: The Magic Feedback Loop (30 seconds)

After the AI completes a task, you only need to say:
- **"Parfait"** if everything works correctly
- **"Faux"** if something needs fixing

That's it! The AI will learn and improve with each interaction.

## 💡 Pro Tips

1. **Be specific**: "Create a blue button" is better than "make it nice"
2. **Check the examples**: Look in `exemple/` folder for real usage
3. **Trust the process**: Let ODIN fail and learn - it gets better!

## 🆘 Need Help?

- **Something broke?** Just say "Faux" and explain what went wrong
- **Confused?** Check out the [full documentation](docs/PROJECT_PHILOSOPHY.md)
