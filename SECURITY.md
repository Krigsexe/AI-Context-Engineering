# Security Policy

The ODIN framework is designed with security as a core principle. We take the security of this project seriously.

## Supported Versions

The ODIN framework is an evolving set of prompts and methodologies. Security updates and support are provided for the **latest version** of the main prompt available in this repository.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < 5.1.0 | :x:                |

We encourage all users to adopt the latest version of the `prompts/ODIN.md` prompt for the most up-to-date features and security considerations.

## Reporting a Vulnerability

We appreciate your efforts to responsibly disclose your findings, and we will make every effort to acknowledge your contributions.

To report a security vulnerability, please create a [GitHub issue](https://github.com/Krigsexe/AI-Context-Engineering/issues) in this repository.

Please include the following details in your report:
- A description of the vulnerability and its potential impact.
- The version of the prompt you are using.
- Steps to reproduce the issue, if applicable.
- Any potential mitigations you might have considered.

Since this is a prompt engineering project, we are particularly interested in vulnerabilities related to:
- Prompt injection attacks.
- Unsafe or malicious code generation.
- Bypassing the core security rules defined in the prompts.
- Information leaks or privacy violations.

Once a vulnerability is reported, we will review it and you can expect an update within 7 days. If the vulnerability is accepted, we will work on a patch and release a new version of the prompt.
